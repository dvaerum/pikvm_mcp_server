# iPad Collector

Native SwiftUI app that pairs with `bench-collect-synthetic.ts` /
`bench-collect-trajectory.ts` on the Mac to generate auto-labeled
training data (no human labelling required) and to record cursor
trajectories for pointer-acceleration modelling.

This directory contains **only the Swift source files**. The Xcode
project file (`.xcodeproj`) is generated by you in Xcode — instructions
below.

## What's where

```
ipad-collector/
├── README.md                   ← this file
├── Sources/
│   ├── iPadCollectorApp.swift  ← @main entry + SessionStore
│   ├── RootView.swift          ← full-screen view + settings chip
│   ├── PointerTracker.swift    ← .onContinuousHover wrapper
│   ├── WebSocketClient.swift   ← protocol + RPC + streaming
│   ├── SceneRenderer.swift     ← Image / Procedural / Video renderers
│   └── SettingsView.swift      ← collector-URL config sheet
└── Resources/                  ← (empty, for future video assets etc.)
```

## One-time setup in Xcode (~5 min)

1. **Open Xcode.** File → New → Project → iOS → **App** → Next.
2. **Configure target:**
   - Product Name: `iPadCollector`
   - Team: your free Apple ID
   - Organization Identifier: anything unique (e.g. `com.yourname.ipadcollector`)
   - Interface: **SwiftUI**
   - Language: **Swift**
   - Storage: None
   - Tests: leave off for now
3. **Save the project anywhere convenient.** Suggest a location *outside*
   this repo so the `.xcodeproj` doesn't fight with `.gitignore`.
4. **Remove the placeholder files** Xcode generated (`ContentView.swift`,
   `iPadCollectorApp.swift`, the App's `.entitlements` if any).
5. **Drag the entire `ipad-collector/Sources/` folder** from this repo into
   the Xcode project navigator. Tick "Copy items if needed" only if you
   want a snapshot; for working in-place from this repo, leave it
   unticked (use folder references).
6. **Target settings** (project navigator → top-level project → your
   target → General/Info tabs):
   - Deployment target: **iPadOS 16.0** (for `.onContinuousHover` etc.)
   - Device family: **iPad** only
   - `UIRequiresFullScreen` → **YES** (Info tab → Custom iOS Target
     Properties — add the key if missing)
   - `UIStatusBarHidden` → **YES** (also in Info; status bar stays
     hidden globally even if our SwiftUI `.statusBarHidden(true)` is
     somehow ignored)
   - Background modes: none needed
   - Supported orientations: pick portrait + landscape per your test
     plan (start with portrait only)
7. **Build & Run** (⌘R) with your iPad selected as the target. First
   launch will fail to install — go to **iPad Settings → General →
   VPN & Device Management** and trust your Apple ID developer cert.
   Re-run.

## Running a session

1. Start the Mac collector:

   ```sh
   npx tsx bench-collect-synthetic.ts --target 100
   # …or…
   npx tsx bench-collect-trajectory.ts
   ```

   It binds `ws://0.0.0.0:8767`. Note your Mac's LAN IP (`ipconfig getifaddr en0`).

2. Launch the iPad app. On first launch the settings sheet opens — enter
   `ws://<your-mac-ip>:8767` and tap **Connect**. The chip in the top-right
   turns green.

3. *(Recommended)* Enable **Guided Access** before unattended runs:
   - iPad Settings → Accessibility → Guided Access → On
   - Triple-click the side button to lock into the app; disables Control
     Center / Notification Center swipes that PiKVM cursor moves could
     accidentally invoke.

4. The collector drives PiKVM HID, requests scenes, asks the app where
   the cursor is, saves labeled frames. Watch progress in the Mac
   terminal.

## Updating the source files

If you keep the source files as folder references (step 5 above), edits
made in this repo's `ipad-collector/Sources/` show up in Xcode
immediately. Hit ⌘R to rebuild.

Apple's free Apple ID signing expires every **7 days**. Re-run via Xcode
to refresh the cert; the app keeps your settings in `UserDefaults`.

## Troubleshooting

- **App can't reach the Mac.** Check the iPad is on the same Wi-Fi as
  the Mac (PiKVM's LAN is fine too if both are on it). Confirm the
  collector is bound to `0.0.0.0`, not `127.0.0.1`.
- **Cursor not reporting.** Make sure the cursor is actually moving on
  the iPad screen (a connected trackpad / Magic Mouse, or PiKVM HID).
  `.onContinuousHover` requires an indirect pointer; finger touches do
  *not* fire it.
- **Frames look black.** The "blackHoldingPattern" scene is the default
  before the collector pushes anything. Confirm the collector logs
  `show-scene → ack` and the iPad app shows the green status chip.
